Aurora · Weather and Climate Architecture

A 1.3-billion-parameter 3D Swin Transformer with Perceiver-based atmospheric encoders and decoders.

Aurora is a single foundation model that ingests heterogeneous Earth-system fields — surface, multi-level atmospheric, static and source-specific variables — lifts them into a universal 3D latent representation, evolves that state in time with a U-Net Swin Transformer, and decodes back to forecasts at arbitrary resolution. The same pretrained backbone is fine-tuned for high-resolution weather, air quality, ocean waves and tropical-cyclone tracks.

02 · Architecture

End-to-end model map

Native multi-source Earth-system fields → universal 3D latent → predicted state at t + Δt, recursively rolled out for longer horizons.

Aurora has three parts: an encoder that converts heterogeneous inputs into a universal 3D latent representation, a processor that evolves that representation in time, and a decoder that translates the latent state back into specific predictions. arxiv 2405.13063 The processor is a 3D Swin Transformer; the encoder and decoder are Perceiver-based modules. 04 · Encoder

3D Perceiver encoder · how heterogeneous inputs become a universal latent

The encoder is what makes Aurora a foundation model rather than a single-dataset forecaster: a Perceiver lets it accept inputs whose resolution, variables and pressure-level count differ from training pass to training pass.
Step a
Patch embed

Input fields are sliced into P×P spatial patches and projected into token embeddings with pressure-level-specific linear weights per variable, so the encoder can specialise representations by altitude.

arxiv 2405.13063
Step b
Perceiver level aggregator

Pressure-level tokens cross-attend to a small set of latent levels. This collapses a variable, per-dataset number of physical levels into a fixed latent depth, so the processor sees a uniform 3D tensor regardless of input source.

latent shape: (H/P, W/P, L_latent, D_embed)
Step c
Additive Fourier encodings

Three additive encodings are concatenated to every token: patch position, patch area and absolute time. The patch-area encoding is what lets a single trained Aurora switch between 0.1°, 0.25° and 0.4° outputs.

i

Why a Perceiver here

A flat ViT would require fixed shapes; Aurora's pretraining mixes ERA5, CMIP6 simulations (CMCC, IFS-HR), operational GFS analyses — each with different resolutions and pressure-level conventions. Perceiver cross-attention is the mechanism Microsoft Research credits for letting one set of weights ingest all of them.

05 · Processor

3D Swin U-Net · the time-evolution backbone

A U-Net of 3D Swin Transformer blocks with shifted local windows. Halves spatial resolution and doubles embedding dim at each merge stage, then symmetrically splits back up.
U-Net layer schedule
48 layers total
Mechanism
Shifted-window 3D self-attention
  • Window size(depth, width, height) = (2, 4, 2) tokens
  • Window shiftingWindows are shifted every other layer — provides information exchange between neighbouring regions
  • Patch mergingHalves spatial resolution of the 3D tensor; embedding dim is doubled in compensation
  • Patch splittingDoubles spatial resolution back; embedding dim halves
  • Multi-scaleLets the backbone simulate the underlying physics at multiple scales

Every other Swin layer shifts the partition by half a window, so tokens at window borders join a new window at layer i+1 — the mechanism that lets local attention propagate information globally.

06 · Decoder & rollout

From latent back to physical fields, autoregressively

The decoder mirrors the encoder: de-aggregate latent levels, then dynamically construct per-variable patch reconstructions.
Decoder path
3D Perceiver decoder
  • a · Level de-aggregatorPerceiver-style cross-attention from L_latent back to the requested physical pressure levels L_phys
  • b · Patch reconstructionEach P×P spatial patch is reconstructed via variable-specific linear layers whose weights are dynamically constructed per output variable
  • c · Air-quality branchFor CAMS, a separate 3D Perceiver decoder learns distinct features for sparse, skewed pollutant fields
  • NumericsComputations before and after the backbone run in 32-bit float for sparse, low-magnitude pollutant values
Forecast rollout
Recursive autoregression

Aurora produces a forecast for a single lead time per forward pass. Longer-lead forecasts are produced by recursively feeding predictions back into the model as inputs. arxiv 2405.13063

07 · Training

Pretraining and fine-tuning

A single 1.3B-parameter set of weights is pretrained on heterogeneous Earth-system data, then fine-tuned twice per downstream domain — first for short lead times, then for long-lead rollouts using LoRA.
01 · Pretraining
Diverse Earth-system corpus

Trained on more than one million hours of diverse geophysical data drawn from climate simulations, reanalysis products and operational forecasts. The objective is a forecast loss summed over multiple heterogeneous datasets with different resolutions, variables and pressure levels.

  • ERA5Reanalysis · primary single-dataset baseline
  • CMIP6 (CMCC, IFS-HR)Lower-fidelity climate simulations · broaden distribution
  • GFS analysesOperational analysis data · covers full atmosphere best
  • ObjectivePer-target weighted loss across all datasets simultaneously
02 · Short-lead fine-tune
All weights, single-step

The full pretrained model is fine-tuned for a target domain (e.g. high-resolution weather at 0.1°, air quality, ocean waves) on its native data, optimising a single-step forecast loss.

MS Research Nature 2025
03 · Long-lead fine-tune
LoRA rollout fine-tuning

Long-lead-time fine-tuning uses Low-Rank Adaptation (LoRA) applied on top of the short-lead-tuned weights. Forecast is rolled out across multiple steps and the loss is back-propagated through the unrolled chain.

LoRA · rank-r adapters backbone frozen except for LoRA
Scaling observations (paper)
Data & size both move the needle
  • Data diversityAdding CMIP6 (C2) lowers RMSE almost uniformly vs ERA5-only (C1)
  • More sourcesAdding more simulation data + analyses (C3, C4) improves on most surface vars and pressure levels covered
  • Model scalingValidation loss drops as a power law — reported as roughly 5% reduction per doubling of model size for fixed GPU-hours
  • Reported onERA5 2021, 6 h lead time, normalised vs ERA5-pretrained baseline MS Research blog

10 · Open phase, Nov 2025

The next phase: open and collaborative AI

From the Microsoft On the Issues post, 13 November 2025: Aurora's source code and model weights are already open; the next phase puts training pipelines, future model releases and new models built on Aurora in the open.
Openness
Code + weights open

"Aurora's source code and model weights are already open" — the announcement extends this to future releases and training pipelines. Exact license text and checkpoint URLs are not reported in the listed Microsoft sources on this page.

Stewardship
Microsoft × Cambridge

Continued development is being run in partnership with Professor Rich Turner's group at the University of Cambridge, through a Microsoft AI for Good grant plus Microsoft research scientists. Originally developed by Microsoft Research AI for Science.

Microsoft frames priority deployment as enabling national meteorological services — particularly across the Global South — to adapt, extend and innovate on top of Aurora for their local forecasting systems.

11 · Caveats

Unknowns & things to verify

Items that are either not reported in the listed sources for this page, or that require checking against the Nature paper Methods and supplementary material.
?

Exact pressure-level set

The Microsoft blog says Aurora ingests a variable number of physical pressure levels and collapses them to a fixed latent count. The exact list of physical levels used at training (e.g. 13 ERA5 pressure levels) and the size of L_latent need to be checked against the Nature 2025 paper. not reported here

?

Patch size P, embed dim D, window dims

The (2, 4, 2) 3D self-attention window is reported. Spatial patch size P and embedding dim D per stage need verification from the paper. verify

?

Rollout horizon

Listed Microsoft sources describe 5-day air-quality forecasts and 5-day-plus medium-range weather forecasts; the long-lead LoRA fine-tune schedule and the maximum supported horizon are not reported on this page and should be read from the Nature paper. verify

?

Released checkpoint URLs & license text

The November 2025 blog post says "Aurora's source code and model weights are already open" and the Azure AI Foundry Labs project page lists Aurora. The exact open-source license, checkpoint download URLs and which variants are released as weights vs code-only must be verified at the official repository and Foundry Labs page. verify before quoting

i

Speedup figure: 5,000x vs ~50,000x

The Microsoft Research blog estimates a ~5,000x speedup over IFS. Secondary coverage of the Nature paper has reported "up to 50,000x faster on a single GPU." Both numbers are source-reported and the framing differs (vs IFS vs single-GPU). They are not contradictory but not interchangeable; use whichever the cited source uses.

!

Do not infer mechanisms not in sources

Aurora is not a diffusion model, not an ensemble model in the Nature paper as described (deterministic single forecast per init), and not trained with RLHF, retrieval, or distillation according to the listed sources. Do not annotate it with mechanisms it does not have.
