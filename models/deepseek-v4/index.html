DeepSeek-V4: million-token reasoning at a tenth of the KV cache.

A pair of Mixture-of-Experts language models — DeepSeek-V4-Pro (1.6T params, 49B active) and DeepSeek-V4-Flash (284B params, 13B active) — that natively support a one-million-token context. This page visualizes the architecture: hybrid attention (CSA+HCA), manifold-constrained hyper-connections, the DeepSeekMoE feed-forward, and the pre/post-training pipeline that produces them.

02 · End-to-end architecture

The transformer block, unrolled.

L stacked blocks. Inside each: an attention sublayer (CSA or HCA) and a DeepSeekMoE feed-forward sublayer, both wrapped in mHC residual mixing. Embeddings flow up; the prediction head emits the next token; an MTP module emits the second.

Reading the block

Compared to a vanilla pre-norm transformer, every "+" residual junction is replaced with three matrices — Al selects an input channel from the widened residual, Bl mixes it through a doubly-stochastic transform, Cl writes the output back. This applies symmetrically around both the attention and FFN sublayers.

The attention sublayer alternates two regimes: CSA layers run sparse attention over moderately-compressed KV; HCA layers run dense attention over heavily-compressed KV. The first two blocks of Flash are pure sliding-window; Pro's first two are HCA.

  • layers (Pro)61 transformer blocks · d=7168 · 128 query heads
  • layers (Flash)43 transformer blocks · d=4096 · 64 query heads
  • attentionCSA ⇄ HCA interleaved, plus 128-token sliding window everywhere
  • MoE1 shared + 384 routed experts (Pro) / 256 routed (Flash); top-6 routed activated
  • MoE first-3Hash routing — input-token-ID determines expert (no learned router)
  • MTP depth1 — predict token t+2 from the second head
  • mHC widthnhc = 4 channels of width d

03 · Residuals, upgraded

Manifold-Constrained Hyper-Connections.

Instead of one residual stream of width d, mHC carries nhc=4 parallel streams and mixes them through a doubly-stochastic matrix at every block. The constraint bounds the spectral norm to 1 — propagation is non-expansive and stable across hundreds of layers.

The four matrices per layer

  • A_l1 × nhc input selector. Sigmoid'd to be non-negative & bounded. Chooses which residual channel feeds the layer.
  • B_lnhc × nhc residual mixer. Projected onto the Birkhoff polytope via 20 Sinkhorn-Knopp row/col normalizations. Guarantees stability across deep stacks.
  • C_lnhc × 1 output writer. Writes the layer output back into the widened residual.
  • paramsEach of A/B/C is dynamic — generated from RMSNorm(vec(X_l)) by a learned projection — plus a static learnable bias. Small α gating factor at init.
Why it matters. Naive Hyper-Connections (Zhu et al., 2025) were unstable when stacked. The doubly-stochastic projection on B_l + Sigmoid clamps on A_l, C_l are what makes the technique scale to 61-layer trillion-parameter training without divergence.
04 · Where the long-context budget goes

Hybrid attention: two compression regimes, interleaved.

Vanilla attention is quadratic; even MLA in V3 still grows linearly per token in cache. V4 chooses a regime per layer: CSA compresses KV moderately (m=4) and then runs sparse attention; HCA compresses KV heavily (m'=128) and runs dense attention over the tiny budget that remains. Both pair with a small sliding-window branch to preserve local fidelity.

+ Sliding-window branch

128-token uncompressed window attached to both CSA and HCA. Recovers local fidelity that compression would otherwise discard.

+ Attention sink

Per-head learnable sink logit added to the softmax denominator. Lets a query head total attention drop below 1 — useful for "abstain" behavior on uninformative positions.

+ Partial RoPE

Rotary embeddings applied to only the last 64 dims of Q, KV, and the attention output (with reversed position −i) — preserves relative position despite the compressed weighted sums.
05 · Compressed Sparse Attention

CSA: compress to ¼, then select top-k.

CSA does two things back-to-back. First, every m=4 consecutive KV entries are merged into one via softmax-weighted compression. Second, a lightning indexer scores each compressed entry against the query and the dense attention layer sees only the top-k.

Why "lightning" indexer?

Picking top-k can't itself be expensive. The indexer uses low-rank queries (down/up projection from the same latent cQ_t shared with main attention), 64 cheap heads, ReLU instead of softmax, and runs entirely in FP4. 2× faster than a naive scorer at 99.7% recall.

Why overlap in compression?

Each compressed entry sums over 2m tokens drawn from two streams (Ca, Cb) with a softmax across all 2m. Effective rate is still 1/m but adjacent blocks see overlapping context — like a stride-1 convolution over a stride-m max-pool.
06 · Heavily Compressed Attention

HCA: 128:1 compression, then attend densely.

HCA strips out the indexer entirely. It compresses 128 consecutive tokens into one entry — non-overlapping — and runs ordinary dense multi-query attention over the few thousand entries that remain even at 1M context. No top-k, no sparse mask. Computational complexity becomes linear-ish.

Why both? HCA alone would discard too much detail; CSA alone makes early layers (which need broad locality) too sparse. Interleaving lets some layers compress aggressively (HCA) and others compress moderately but selectively (CSA). Empirically this beats either pure strategy at the same total FLOPs.
07 · The feed-forward, sparsified

DeepSeekMoE: 1 shared + 384 routed experts.

Every block's FFN is replaced by a Mixture-of-Experts. For each token, a router selects top-6 routed experts; their outputs sum with one always-on shared expert. The first three MoE layers skip the learned router entirely — token ID hashes directly to expert.

Sqrt(Softplus) affinity

V3's router used Sigmoid. V4 swaps to sqrt(softplus(x)) — preserves monotonicity and bounded growth but doesn't saturate to 1, so the relative ordering of experts is preserved even at high logits.

Aux-loss-free balance

No auxiliary load-balancing loss on most steps — a small per-expert bias is updated at 0.001 rate to nudge router toward uniform usage. A light sequence-wise balance loss (weight 0.0001) prevents extreme single-sequence skew.

Unbounded routing nodes

V3 capped how many GPU nodes a token could route to. V4 lifts the cap entirely — the parallelism scheme handles arbitrary fan-out, with fine-grained communication-computation overlap inside the MegaMoE kernel.
08 · The optimizer

Muon for the bulk, AdamW for the edges.

Muon updates almost every weight matrix: it orthogonalizes the momentum-buffered gradient via hybrid Newton-Schulz iterations before applying it. AdamW is retained only for embeddings, the prediction head, mHC static biases & gating factors, and RMSNorm weights.

Why Muon

Orthogonalizing the update matrix keeps every direction equally weighted — gradients with vastly different singular-value scales don't collapse the update into a low-rank approximation. In trillion-parameter MoE, that low-rank failure mode dominates.

09 · Pre-training

32T+ tokens, sequences grown from 4K to 1M.

Both models are trained on 32-33T tokens of mixed code, math, web, and long-document data. Context length is scheduled from 4K up to 1M; attention starts dense and switches to sparse mid-training.

Stability tricks discovered along the way

Anticipatory Routing

Loss spikes were tied to MoE outliers, and the router exacerbated them. Fix: compute routing decisions at step t using parameters from step t − Δt. Decouples backbone and router updates; auto-triggered only on spike detection. ~20% extra wall-clock when active.

SwiGLU Clamping

Linear component of SwiGLU clamped to [−10, 10]; gate's upper bound capped at 10. Effective at eliminating activation outliers with no measured quality cost. Applied throughout both V4-Pro and V4-Flash training.
10 · Post-training

Train ten specialists, distill into one.

Post-training drops the mixed-RL stage entirely in favor of On-Policy Distillation. Domain experts are trained independently with SFT + GRPO; then a single student model learns from all of them by minimizing reverse-KL over its own rollouts.

Reasoning modes

Non-think

Direct response, no reasoning tokens. Output starts with </think> immediately. For routine tasks where fast TTFT matters more than depth.

Think-High

Standard reasoning trace inside <think>…</think>. Context window 128K. Most common production mode.

Think-Max

Maximum reasoning effort. Special "no shortcuts permitted" system prompt prepended. 384K context. Used to define V4-Pro-Max headline numbers.

FP4 quantization-aware training

Two components quantize to MXFP4 during the QAT phase: MoE expert weights (the bulk of GPU memory), and the indexer QK path in CSA. FP4 → FP8 dequantization is lossless thanks to FP8's two extra exponent bits absorbing the per-tile scale. Index scores additionally quantize FP32 → BF16, giving 2× speedup at 99.7% top-k recall.

11 · Training & inference infra

The system layer that makes the architecture practical.

A trillion-parameter MoE with hybrid sparse attention at 1M context is a software problem first. The report calls out six load-bearing infrastructure pieces.

MegaMoE kernel

Single fused CUDA kernel that overlaps dispatch all-to-all, GEMM-1, SwiGLU, GEMM-2, and combine all-to-all by splitting experts into waves. 1.5-1.7× over non-fused, 1.96× on RL rollouts. Open-sourced in DeepGEMM.
1.92× theoretical

TileLang DSL

Replaces hundreds of Torch ATen ops with fused kernels. Host-side codegen via TVM-FFI drops per-invocation overhead from ~100µs to <1µs. Z3-backed integer reasoning unlocks more aggressive vectorization passes.

Batch-invariant kernels

Bitwise-identical outputs regardless of a token's position in batch — needed for training/inference alignment. Dual-kernel attention strategy; DeepGEMM replaces cuBLAS; deterministic backward via per-SM accumulation buffers.

Heterogeneous KV cache

Two-tier layout: classical KV cache for CSA/HCA compressed entries (block size = lcm(m,m')), state cache for SWA + uncompressed tail tokens. On-disk storage for shared-prefix reuse with three SWA strategies (full/checkpoint/zero).

Contextual parallelism

Two-stage CP for compressed attention: rank i sends its last m uncompressed entries to rank i+1, then an all-gather reconstructs the full compressed sequence. Handles cross-rank compression boundaries.

DSec sandbox

Production sandbox platform for agent training/eval. Four substrates (Function Call / Container / microVM / fullVM) behind one Python SDK. EROFS on-demand image loading; trajectory log enables fast-forward resumption after GPU preemption.
