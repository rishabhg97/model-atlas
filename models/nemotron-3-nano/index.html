02 · Layer pattern

The 52-layer stack, decoded

Nemotron 3 Nano interleaves three primitive layer types in a fixed, repeating motif. The pattern follows the previous Nemotron generations but every FFN slot is replaced with a granular MoE layer. We use a hybrid Mamba-Transformer architecture similar to the previous generation of Nemotron models. In addition, we scale the model sparsely by using MoE layers instead of standard FFN layers.

03 · Primitive layers

Three building blocks, one residual stream

Each layer takes the residual stream of width 2 688, applies an RMSNorm, runs its primitive, and adds back. No layer changes the hidden dim. The three primitives differ in what they mix — tokens (sequence), heads (channels), or experts (capacity).

04 · MoE in detail

6 of 128 routed + 2 always-on shared

Each MoE layer holds 130 expert MLPs. The router scores all 128 routed experts and keeps the top-6; the 2 shared experts run unconditionally. To achieve the best accuracy, we use a granular MoE architecture along with shared experts. For the MoE layers, we use squared ReLU activation and a standard learnt MLP router with sigmoid gating.

06 · Pre & post processing

What sits outside the layer stack

Three deliberate omissions — no positional encoding, no dropout, no biases on linear layers — keep both ends of the model unusually thin.

Pre-processing — input side

Tokens → residual stream

  • Tokenizer. BPE tokenizer shared with the released checkpoints; produces integer token ids of shape [B, T].
  • Input embedding. A learned table E ∈ ℝ^(V × 2688) projects each token id into the residual stream. Untied from the LM head, so reading and writing the vocabulary use independent matrices.
  • No positional encoding. The model is run with NoPE; positional information emerges from the causal Mamba-2 recurrence and the causal attention mask.
  • No dropout. Removed throughout — quoted directly from the architecture description.
  • No bias on linear layers. Every W · x in the stack omits its b.
Post-processing — output side

Residual stream → logits

  • Final RMSNorm. A single root-mean-square norm with learned scale γ is applied to the last residual vector — no bias term, no centering.
  • LM head. A linear projection W_out ∈ ℝ^(2688 × V) emits logits of shape [B, T, V]. Not tied to the input embedding.
  • Sampling. External to the model — temperature, top-p, and (in reasoning mode) the chat template's reasoning channel are applied by the runtime.
  • Chat template. A reasoning-aware template wraps user / assistant / tool turns. In multi-turn mode, prior-turn reasoning traces are dropped from the prompt; in multi-step mode within a single turn they are preserved.
  • Tool calls use XML-style special tags rather than JSON-escaped strings to reduce character escaping.

Pre-training · 25 T tokens

Warmup-Stable-Decay LR over 25 T tokens of 15-category text. Phase 1 promotes diversity (web crawl, code, math, multilingual, synthetic SFT-style); Phase 2 narrows to high-quality at the 94 % mark. Sequence length 8 192, batch 3 072 (~25 M tokens/batch).

LC-phase · 121 B tokens

Continuous pre-training to 512 k tokens. Mix of 512 k and 4 k sequences with a small retrieval-focused synthetic blend; 8-way context · 8-way tensor · 8-way expert · 4-way pipeline parallelism on H100.

SFT · 18 M samples

Chat-template applied. Domain mix: chat 28.6%, code 20.7%, science 12.8%, math 9.9%, multilingual 7.4%, math+tools 4.9%, SWE 3%, GenSelect 3%. 13 k steps, batch 64, sequences packed to 256 k. Reasoning on/off via 10% strip, budget control via 3% truncate.

RLVR · multi-environment

Synchronous GRPO with masked importance sampling. 128 prompts × 16 generations, batch 2 048, on-policy. Eight environments trained simultaneously — math, code, QA, structured output, instruction following, long context, two agentic tool-use. MoE router weights frozen; aux-loss-free bias still updated.

RLHF · GenRM + length control

Qwen3-235B GenRM scores pairs of completions with helpfulness + ranking heads. Circular comparison cuts O(N²) to O(N). A group-relative length bonus + quality-gated conciseness bonus cuts verbosity ~30% without losing accuracy. A second RLVR pass follows.

PTQ · BF16 → FP8

ModelOpt + Megatron-LM. 1 k calibration samples from the post-training SFT blend. Selective: 6 attention layers + 6 attention-feeder Mamba layers + all Conv1Ds stay BF16; the rest of weights, activations, and KV cache go FP8. ~99% median accuracy recovery, ~3.4× throughput.
