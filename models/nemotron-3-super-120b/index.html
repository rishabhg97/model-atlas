02 · End-to-end map

From a token id to a logit vector.

The model is a single residual stream of width d = 4 096. Tokens enter the left edge as integers, traverse the 88-layer hybrid stack, and exit as a logit vector over 131 072 vocab entries. Pre- and post-processing are deliberately thin: no positional embedding, no dropout, no bias on linear layers, RMSNorm throughout, and the LM head is untied from the input embedding.

03 · Hybrid stack

The 88-layer interleave: 40 Mamba-2, 40 LatentMoE, 8 attention anchors.

Sequence mixing is carried by Mamba-2 — linear-time, constant generation state — and channel mixing by LatentMoE. Eight sparse attention anchors punctuate the stack to preserve global token interaction and long-range routing without paying the quadratic attention cost in every layer. The pattern below is taken from the technical report (Figure 2) and the NVFP4 checkpoint config; the meaning of * as a sparse / global-attention anchor matches the report's narrative, but the precise implementation should be cross-checked against source.

04 · Attention anchor

Grouped-query attention, sparingly placed.

Where attention does run, it runs as GQA with a 16:1 query-to-KV ratio — 32 query heads, 2 KV heads, head dimension 128. The model omits positional embeddings entirely; position is implicit in the causal Mamba-2 recurrence and the causal attention mask. Anchors are sparse so the stack avoids quadratic attention cost in every layer.

Layer kind · 8 of 88

Global attention anchor · GQA

16:1 head sharing. 32 query heads project from the residual stream; only 2 KV heads are stored, each shared across a contiguous group of 16 Q heads. This shrinks the KV cache by 16× vs. multi-head attention without losing query-side resolution.

Anchor placement. Eight attention layers across 88 — roughly one anchor per 11-layer stretch — give the stack a periodic full-token interaction surface. Most layers route information through Mamba-2's recurrent state instead.

No positional encoding. Sources do not say RoPE is used; the model is described as omitting positional embeddings. Position comes from the causal recurrence and mask.

05 · Mamba-2

Linear-time sequence mixing, constant generation state.

The 40 Mamba-2 layers do most of the sequence-mixing work. Each layer projects the residual stream into a wider lane, applies a short causal convolution, runs a selective state-space scan with state size N = 128 over 128 parallel heads, gates the output, and projects back. During autoregressive generation the per-layer state is a fixed-size tensor — independent of sequence length — which is what makes 1 M-token contexts tractable.

Layer kind · 40 of 88

Mamba-2 SSD block

Selective state-space duality (SSD). A second-generation Mamba block: the recurrence and a chunked matrix form are dual views of the same operator, so training packs sequences into chunks of 128 tokens while inference runs the recurrence step-by-step.

Why it scales. Per-layer memory at generation time is the SSD state, not a KV cache that grows with T. That's the property that makes long-context reasoning affordable.

Mixed precision in NVFP4. The Mamba output projection is one of the layers the report places in MXFP8 rather than NVFP4 — see the precision map.

06 · LatentMoE

Move the experts into latent space.

The defining mechanism of Nemotron 3 Super. Each LatentMoE layer routes 22 of 512 experts per token — but unlike a standard MoE the routed expert MLPs operate on a 1 024-dimensional latent projection of the residual stream, not on the full 4 096-d hidden state. A pair of shared down/up projections (W_down, W_up) bracket the routed compute, while the router, the one shared expert, and the rest of the layer remain in full hidden dimension. The all-to-all payload and the per-token expert FLOPs both shrink by roughly d / l ≈ 4×, which is what lets the model push to 512 experts and top-22 at serving cost comparable to a much smaller MoE.

Layer kind · 40 of 88 · routed top-22 of 512 + 1 shared

LatentMoE block

Compute happens in ℝ^1024. The down-projection W_down brings the 4 096-d token vector into a 1 024-d latent. The 22 routed experts read and write that latent. W_up projects the aggregated result back to 4 096-d before the residual add.

Router and shared expert stay full-width. The router scores all 512 experts from x ∈ ℝ^4096; the single shared expert is computed unconditionally in full hidden dimension and added to the routed result.

Why this works. Per-token expert FLOPs and the all-to-all transfer between expert-parallel ranks both scale with l rather than d. At serving cost roughly comparable to a same-active-params MoE in 4 096-d, the model fits more experts (512) and selects a larger top-k (22), giving sparser specialization per token.

07 · Multi-token prediction

Predict the next several tokens, then verify them in parallel.

A shared-weight MTP head sits next to the LM head and predicts tokens at multiple future offsets. During training MTP is a representation-shaping auxiliary loss; at inference it acts as a built-in draft model, recursively applied to produce a length-k draft that the main model then verifies in a single forward pass — no external draft network required. Source caveat: the technical report describes 2 shared-weight MTP layers; the NVFP4 checkpoint config exposes num_nextn_predict_layers = 1. Treat this as a report vs. checkpoint-config difference, not a contradiction.

08 · NVFP4 pretraining

4-bit linears, BF16 spine, full-horizon training.

Nemotron 3 Super was pretrained end-to-end in NVFP4 — not a post-training quantization. Linear layers run in a 4-bit float format with 16-element micro-blocks and a small E4M3 scale per block; the report keeps quant-sensitive paths (latent projections, MTP, attention projections, embedding, the final 15 % of the network) in BF16, and places the Mamba output projection in MXFP8. The report also investigated MXFP8 "healing" of NVFP4 checkpoints, but did not find sustained downstream gains.

NVFP4 training recipe — at a glance

  • Element formatE2M1 — 1 sign, 2 exponent, 1 mantissa bit. The 4-bit float used for weights and activations on NVFP4 paths.
  • Micro-block16 elements share one block-level scale, applied along the reduction axis to keep dynamic range usable.
  • Block scale formatE4M3 — a small 8-bit float, low overhead, sufficient range for per-block normalization.
  • Global scaleFP32 per tensor — coarse, high-precision rescaling so that fine-grained E4M3 blocks land in their happy range.
  • RoundingStochastic rounding on quantization to keep gradient estimates unbiased over training.
  • Hadamard mixingRandom Hadamard transforms on some gradient paths spread per-element magnitude and reduce outliers before quantization.
  • Training horizonNVFP4 used for the full pretraining token horizon — not just a final phase. MXFP8 "healing" was investigated; the report reports no sustained downstream gain.
09 · Long-context training

Extending to 1 M tokens — then verifying it actually works.

After the main pretraining run, a continuous-pretraining phase extends the sequence length to 1 048 576 tokens, followed by an alternating 1 M / 4 K phase to recover short-context behavior. SFT then packs sequences to 256 K, with a later long-context stage at 512 K including synthetic multi-document and multi-hop QA data.

10 · Training & post-training

Pretrain → long-context → SFT → RLVR → SWE-RL → RLHF → MTP healing.

25 T+ pretraining tokens of crawled and synthetic text covering code, math, science, and general knowledge. Long-context extension. SFT on 7 M+ samples / ~80 B tokens. Then multi-environment RL from verifiable rewards (NIM card summarizes this as asynchronous GRPO across math, code, science, instruction following, tool use, multi-turn conversations, structured output), a software-engineering RL stage, RLHF for interaction quality, and a final MTP-only healing pass with the main weights frozen.

12 · Caveats & unknowns

Where the report, the config, and the cards disagree.

This page sticks to source-backed numbers. Where sources differ in level of detail, both versions are labeled rather than silently merged.

Parameter counts

Product cards round to 120 B total / 12 B active. The technical report gives a more precise 120.6 B total / 12.7 B active (12.1 B excluding embeddings). Both rows are accurate at their stated precision.

Context length depends on deployment

The technical report and model cards say up to 1 M tokens. The current NVFP4 config.json has max_position_embeddings = 262 144, and the NIM quickstart defaults to 256 K unless configured for 1 M. Treat as a runtime/config caveat, not a contradiction.

MTP layer count

Technical report describes 2 shared-weight MTP layers. The NVFP4 config exposes num_nextn_predict_layers = 1. Report vs. checkpoint-config difference; the inference path is the same shared-weight head applied recursively.

Attention-anchor implementation

Sources describe sparse / global attention layers punctuating the Mamba-2 stack; the * marker in the layer pattern is interpreted here as an attention anchor based on the report narrative. Confirm against the checkpoint's exact layer_types array before depending on it in implementation.

Positional encoding

Sources state the model omits positional embeddings. No source explicitly supports RoPE for Nemotron 3 Super, so this page does not claim it.
