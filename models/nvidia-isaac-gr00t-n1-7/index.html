NVIDIA Isaac GR00T N1.7 Reasoning VLA Architecture

An open, commercially-licensed Vision-Language-Action foundation model for humanoid robots that maps RGB observations and natural-language instructions to continuous motor commands via a dual-system Action Cascade: a Cosmos-Reason2-2B VLM for high-level reasoning, and a 32-layer Diffusion Transformer for real-time motor control. Released by NVIDIA on April 17, 2026. [sources]

Lineage from N1 → N1.6 → N1.7

GR00T N1 (Mar 2025) introduced the dual-system VLA trained on egocentric human video, real and simulated robot trajectories, and synthetic data, validated on Fourier GR-1 and 1X humanoids[3]. N1.7 retains the dual-system design and upgrades the VLM backbone to Cosmos-Reason2-2B, adds task-level and subtask-level reasoning, expands dexterous manipulation to finger-level 22 DoF control, and replaces teleoperation-scale pre-training with the EgoScale human-video corpus[1].

What is not in this page

§ 01 · End-to-end model map

Observation to action, one step

At inference, GR00T N1.7 consumes a snapshot of the robot's world — one or more RGB frames, a language instruction, and the current proprioceptive state — and emits a continuous-value action vector mapped to the robot's degrees of freedom. The two systems run in cascade: System 2 thinks in tokens; System 1 denoises into joint targets.

§ 02 · Action Cascade

Dual-system VLA, in detail

Action Cascade separates what to do from how to move. The reasoning system runs at a thinking cadence over images and language; the motor system runs at a control cadence over state and action tokens. The split is the same one introduced in GR00T N1 — N1.7 upgrades the VLM and the data, not the topology.

SYSTEM 2 · REASONING

§ 03 · Embodiment adapters

§ 04 · Training

EgoScale — pre-training on human egocentric video

The headline data shift between N1.6 and N1.7 is what the team calls EgoScale: pre-training on 20,854 hours of human egocentric video spanning 20+ task categories, from manufacturing and retail to healthcare and home environments[1]. This is a deliberate move past the few-thousand-hour scale of robot teleoperation that trained N1.6[1].

Corpus scale
20,854 h[1]
Sensorized human egocentric video used for pre-training. More than ten-fold larger than the N1.6 teleoperation corpus[1].

Sensorized recordings — not raw web video — provide manipulation priors that transfer to humanoid embodiments.

§ 05 · Fine-tuning

Adapt to a custom embodiment

N1.7 supports fine-tuning on custom embodiments using the LeRobot dataset format. The HF announcement gives a launch command and notes the upgrade from N1.6 is a drop-in swap of --model-path[1].

Launch command

# Source: HF blog · NVIDIA Isaac GR00T N1.7 [1]
CUDA_VISIBLE_DEVICES=0 uv run python \
  gr00t/experiment/launch_finetune.py \
  --base-model-path      nvidia/GR00T-N1.7 \
  --dataset-path         <YOUR_DATASET_PATH> \
  --embodiment-tag       <YOUR_EMBODIMENT> \
  --modality-config-path <YOUR_MODALITY_CONFIG> \
  --num-gpus             1 \
  --output-dir           <OUTPUT_PATH> \
  --max-steps            2000 \
  --global-batch-size    32

§ 06 · Inference & runtime

From checkpoint to robot loop

Deployment goes through a policy server: load nvidia/GR00T-N1.7 against an embodiment tag, run a control loop against the server with PolicyClient.get_action()[1]. Performance numbers are reported by NVIDIA at 4 denoising steps with a single camera view[1].

Server launch

# 1 — install
git clone --recurse-submodules \
  https://github.com/NVIDIA/Isaac-GR00T
cd Isaac-GR00T
bash scripts/deployment/dgpu/install_deps.sh
source .venv/bin/activate

# 2 — launch policy server
uv run python gr00t/eval/run_gr00t_server.py \
  --embodiment-tag GR1 \
  --model-path     nvidia/GR00T-N1.7

Control loop

from gr00t.policy.server_client import PolicyClient

policy = PolicyClient(host="localhost", port=5555)

obs, info = env.reset()
action, info = policy.get_action(obs)
obs, reward, done, truncated, info = env.step(action)

§ 07 · Unknowns & caveats

What the listed sources do not report

Per the prompt: anything not explicitly grounded in the listed URLs is marked as unknown rather than guessed.

Architecture unknowns
  • VLM hidden dimension, head count, exact Cosmos-Reason2-2B layer count.
  • DiT hidden dimension, head count, action-chunk horizon.
  • Image tokenizer / patch size and total image-token budget per observation.
  • Language context length.
  • Architecture deltas between N1, N1.6, and N1.7 beyond the VLM-backbone swap and EgoScale data.
Training unknowns
  • GPU-hours / compute budget; optimizer, schedule, batch size for pre-training.
  • Robot-trajectory and synthetic-data quantities used in N1.7 (N1 lineage only[3]).
  • Safety filtering and licensing of the underlying egocentric-video corpora.
  • Use (or absence) of RL, distillation, retrieval, or RLHF stages.
Evaluation unknowns
  • Quantitative benchmark scores — no task-success-rate table is given on the HF announcement.
  • Comparison numbers vs. prior generalist robot policies on standard benchmarks.
  • Sim-to-real transfer ablations specific to N1.7.
Source agreement
  • HF blog[1] and N1 research page[3] agree on the dual-system VLA framing and on the role of egocentric human video.
  • Parameter scale: 3 B total is the only count given on the HF announcement[1]; the local backlog row records the model size as "not reported in backlog row", which is consistent (the backlog simply does not carry the figure).
  • No conflicts between listed sources were identified.
