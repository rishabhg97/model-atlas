Alibaba Qwen · sparse hybrid MoE · multimodal (text + image + video)

Qwen3.6 35B-A3B

A 35 B-parameter, 3 B-active sparse Mixture-of-Experts model with a hybrid sequence mixer that interleaves Gated DeltaNet (linear attention with a learned forget gate) and Gated Attention (softmax attention with QK-Norm and partial RoPE) in a strict 3 : 1 ratio across 40 layers. Released April 15, 2026 under Apache 2.0 as Alibaba Qwen's first open-weight Qwen3.6 variant, the model is image-text-to-text by default — a causal LM with a vision encoder — operates in thinking mode by default, supports a 262 K native context extensible to ~1 M tokens with YaRN, and is trained with multi-step Multi-Token Prediction for speculative decoding.

02 · End-to-end map

From multimodal input to text logits and N-step predictions.

A prompt arrives as an interleaved sequence of text tokens, image patches, and video frames. Visual content is encoded by a dedicated vision tower into the language-model token stream; the joint sequence flows through 40 hybrid layers — each layer is either a Gated DeltaNet → MoE or a Gated Attention → MoE block — and exits as logits over the 248 320-entry vocabulary, plus an MTP head that emits N future tokens for speculative decoding.

03 · The 40-layer hybrid stack

Three linear-attention layers, then one softmax-attention layer — repeated ten times.

The model card specifies the layout in a single line: 10 × (3 × (Gated DeltaNet → MoE) → 1 × (Gated Attention → MoE)). Every layer pairs one of two sequence mixers with the same 256-expert MoE channel mixer. 30 of 40 layers use Gated DeltaNet — a linear-attention recurrence that scales as O(T) and carries a compressed state — while every fourth layer is a full softmax-attention layer with M-RoPE and a KV cache. The pattern is fixed, not learned, and identical across all ten repeats.

04 · Gated DeltaNet block

Linear attention with a learned forget gate — three out of every four layers.

Gated DeltaNet replaces softmax attention with a recurrent state that is updated by a delta rule and modulated by an input-dependent gate. The mechanism is linear in sequence length, requires no KV cache, and is the workhorse mixer of Qwen3.6: 30 of 40 layers. The model card pins the head geometry — 32 value heads, 16 QK heads, head dimension 128 — but does not publish the exact update equation; this page reproduces the delta-rule + gate sketch that the Gated DeltaNet name implies and labels the closed-form recipe as not reported.

1 · Pre-mixer RMSNorm
RMSNorm

Pre-norm transformer arrangement: normalize the residual stream before the sequence-mixer sub-block. Standard Qwen-family RMSNorm with a learned scale γ; no bias.

2 · Q, K, V projections
32 V · 16 QK · d_h = 128

Three linear projections produce 16 query heads, 16 key heads, and 32 value heads at head dimension 128. The asymmetric V : QK ratio gives the recurrent state a larger codomain than its dot-product index space — a known Gated DeltaNet design choice.

3 · Forget / output gates
σ(·) · learned

Input-dependent sigmoid gates control (a) how much of the prior recurrent state survives into the next step and (b) how much of the read-out enters the residual stream. Gating is what distinguishes this from a vanilla DeltaNet recurrence.

4 · Recurrent state update
delta rule · linear in T

The state S_t is updated each step by a delta-rule outer product — replacing the value previously associated with the current key — and multiplicatively decayed by the forget gate. No quadratic Q·Kᵀ. No KV cache.

5 · State read-out
o_t = g · (S_t · Q_t)

The current query reads the recurrent state by a matrix-vector product, optionally gated. Output dimension is 32 × 128 = 4 096; W_O projects back to the 2 048-d residual stream.

6 · Residual + MoE sub-block
→ §07

Residual add, then the standard 256-expert MoE channel mixer (top-8 + 1 shared), then another residual. The MoE sub-block is shared by every layer regardless of mixer type — see §07.

05 · Gated Attention block

Full softmax attention with QK-Norm and partial M-RoPE — one in every four layers.

The Gated Attention layers carry the exact long-range routing that linear attention cannot represent. There are 10 of them, one per 4-layer cycle. The model card pins this layer's geometry precisely: 16 query heads, 2 KV heads (an 8 : 1 grouped-query ratio), head dimension 256, and RoPE applied to only the first 64 of those 256 dimensions — a partial-rotary recipe that leaves the remaining 192 dimensions positionless.

1 · Pre-attention RMSNorm
pre-norm

Same RMSNorm as the DeltaNet variant. The Gated Attention layer otherwise has different head counts and projection shapes than its linear-attention neighbour.

2 · GQA 8 : 1 projections
16 Q · 2 KV

16 query heads share 2 KV heads — every KV head is replicated to 8 query heads at attention time. Per-head dimension is 256; the QKV slab is 4 096 + 512 + 512 = 5 120 wide.

3 · Partial M-RoPE on the first 64 dims
rotary_dim = 64

Rotary position embedding rotates only the leading 64 dimensions of each Q/K head; the remaining 192 are positionless. partial_rotary_factor = 0.25 in config.json. The rotation itself is M-RoPE — see §08 for the (t, h, w) interleave.

4 · QK-Norm
per-head RMSNorm

Additional RMSNorm on the per-head Q and K vectors before the dot product. Stabilises attention logits at long context; carried over from the Qwen family. Not stated by name in the fetched model card; carried from inventory lead and Qwen3 conventions.

5 · Softmax attention + output gate
causal · full ctx

Standard scaled-dot-product softmax over the entire causal context. The "Gated" in Gated Attention refers to a multiplicative gate applied to the attention output before W_O — the same gating motif as the DeltaNet layer, giving the two mixers a consistent interface.

6 · MoE channel mixer
→ §07

Identical MoE sub-block to the DeltaNet variant. The two block types share everything except the sequence-mixer module — same norm, same residual structure, same expert bank.

06 · Head geometry · side by side

Two mixers, two completely different projection slabs.

The two sub-blocks differ in head count, head dimension, and the asymmetry between query and value heads. The cards below summarise the model-card numbers verbatim and visualise the per-token state each layer maintains.

07 · Mixture-of-Experts channel mixer

256 experts. Eight routed and one shared active per token. Same module on every layer.

Every layer — DeltaNet or Attention — ends in a single, uniform MoE channel mixer. There are 256 experts per layer; each token routes to its top-8 by router score and additionally pulls from 1 always-on shared expert, for a total of 9 active experts per token per layer. With expert intermediate dimension 512 and 40 layers, this is the mechanism that turns 35 B total parameters into a 3 B activated footprint.

08 · M-RoPE & long context

Three-axis rotary embeddings — time, height, width — applied to a partial-rotary slice.

Qwen3.6 uses Multimodal RoPE (M-RoPE): the rotation budget is split across three position axes — temporal, image-height, image-width — so a single positional encoding scheme covers text, image, and video tokens uniformly. The model card's recommended long-context config sets mrope_section = [11, 11, 10] — 32 rotary pairs split 11/11/10 across (t, h, w) — with mrope_interleaved = true, rope_theta = 10 000 000, and partial_rotary_factor = 0.25, meaning M-RoPE rotates only the first 64 of 256 head dimensions.

The 32 rotary pairs (64 dimensions) form a single rotary slice of each 256-d head; the remaining 192 dimensions per head are positionless. mrope_interleaved = true interleaves the three axes inside that slice rather than stacking them. For pure text, all positions write to the t axis only; for image tokens, h and w become non-trivial; video adds motion along t simultaneously.
Long-context extension

YaRN · static scaling from 262 144 → 1 010 000 tokens

Native vs extended context. The model is trained at 262 144 tokens. YaRN — a learned-rate-free RoPE scaling scheme — can extend the effective window to 1 010 000 tokens by rescaling the rotary frequencies, without further fine-tuning.

Where it lives. YaRN is enabled by overriding rope_parameters in text_config or via inference-server flags (--max-model-len 1010000 on vLLM/SGLang). The override fields are rope_type: "yarn", factor: 4.0, original_max_position_embeddings: 262144.

Why short prompts suffer. Static YaRN applies the same scaling factor everywhere, so 1 K-token prompts get rotated through the same long-range frequencies that were calibrated for 1 M. The model card recommends keeping context at ≥ 128 K to preserve thinking quality if YaRN is on.

09 · Multi-Token Prediction

Trained to predict several future tokens per step — used at inference as a speculative decoder.

The model card lists MTP: trained with multi-steps. In practice the model is shipped with an additional MTP head that emits N future tokens at every position; serving frameworks then use those predictions as draft tokens, verifying them in a single batched forward pass. SGLang and vLLM both expose dedicated flags (--speculative-algo NEXTN, method: qwen3_next_mtp) — the recommended setting is 3 speculative steps · top-1 draft · 4 verify tokens on SGLang.

Why ship MTP weights?
train-time signal

Multi-step MTP training gives the backbone richer gradients (the model must predict several next tokens, not just one), even before any speculative decoding is used. The serving-time speedup is a downstream consequence.

vLLM config
qwen3_next_mtp

--speculative-config '{"method":"qwen3_next_mtp","num_speculative_tokens":2}'. vLLM ships a Qwen3-Next-specific MTP recipe; the speculative head shares the backbone's last-layer hidden states.

SGLang config
NEXTN

--speculative-algo NEXTN --speculative-num-steps 3 --speculative-eagle-topk 1 --speculative-num-draft-tokens 4. Recommended in the model card for the standard 256 K-context deployment.

Acceptance dynamics
not reported

The model card does not publish empirical acceptance rates or end-to-end speedup numbers for MTP on Qwen3.6-35B-A3B. Treat speedup as deployment-dependent.

10 · Vision encoder

An image-text-to-text model — vision is first-class, not bolted on.

The Hugging Face pipeline tag for Qwen3.6-35B-A3B is image-text-to-text: vision is the default input modality alongside text. The model card describes the model as a Causal Language Model with Vision Encoder; the exact vision-tower specification (depth, hidden dim, patch resolution) is not duplicated in the fetched model-card text, but the inference surface is documented: image and video inputs are accepted via the OpenAI Chat Completions API with type: image_url and type: video_url, video frame sampling is driven by an fps parameter, and a --language-model-only flag exists to skip the vision tower entirely when running pure text workloads.

Video input

Driven by fps (default 2) and do_sample_frames. The video processor's longest_edge defaults conservatively; the model card recommends raising it to 469 762 048 (≈ 224 K video tokens) for hour-scale video.

11 · Thinking mode

Reasoning is on by default; preservation across turns is a Qwen3.6 first.

Qwen3.6-35B-A3B "operates in thinking mode by default, generating thinking content signified by <think>…</think> before producing the final response." Unlike Qwen3, the soft /think, /nothink commands are not supported — callers toggle thinking through the chat-template kwarg enable_thinking. The new Qwen3.6 capability is preserve_thinking = true, which retains the reasoning trace from previous turns to streamline iterative agent work.

14 · Caveats & unknowns

What the model card pins down — and what it leaves to the implementation.

The Qwen3.6-35B-A3B model card is unusually precise about architecture geometry (it ships the full layer layout in one line) but stays silent on training scale, data composition, and the closed-form Gated DeltaNet update. The list below catalogues what is not reported in the fetched model-card text and should be sourced elsewhere before quoting.

Gated DeltaNet update equation

The Gated DeltaNet head counts are pinned (32 V / 16 QK · d_h = 128), but the exact state-update recipe — delta-rule formulation, gate parameterisation, normalisation — is not written out on the model card. The diagram in §04 uses the canonical Gated DeltaNet form and labels it as a sketch.

QK-Norm in Gated Attention

QK-Norm is a Qwen3-family convention and is in the Raschka-table entry, but the Qwen3.6 model card does not explicitly re-name it. Treat as inventory-led for now; confirm against config.json on the HF repo.

Positional encoding on DeltaNet layers

M-RoPE is named for the global softmax-attention layers; the positional behaviour of the 30 Gated DeltaNet layers (implicit via recurrence vs explicit rotary on Q/K) is not separately described in the fetched model-card text.

MoE load balancing & auxiliary loss

The model card lists expert counts and intermediate width, but does not describe the load-balancing recipe (auxiliary loss, expert capacity, drop policy) or the router parametrisation. The diagram shows a generic top-8 softmax router as a placeholder.

Vision tower specification

The model is image-text-to-text by design, but the fetched model-card text does not duplicate the vision-tower architecture (depth, hidden dim, patch size). The video preprocessor's longest_edge tuning recipe is the only hard number it ships.

Training data & token budget

Pretraining and post-training stages are listed by name only. No token counts, language distributions, code-share, RL algorithm, reward model, or compute budget are disclosed in the fetched text. The release blog should be consulted for any additional public detail.

Per-layer parameter accounting

The hidden dim (2 048), expert width (512), expert count (256), and layer count (40) are stated — but the model card does not give a parameter-by-component breakdown. The 35 B / 3 B totals are reported; the split between vision tower, MoE experts, attention/DeltaNet projections, and embeddings is not.

Quantisation & serving

The HF repository advertises BF16 tensor type for the base release; over 300 community quantised mirrors exist (none first-party). Quantisation-aware-training details, if any, are not in the model-card text.
