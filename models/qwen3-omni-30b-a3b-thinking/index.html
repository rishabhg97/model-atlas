Open · Omni-modal understanding · Thinker-only checkpoint Chain-of-thought reasoning

Audio, video and text in — reasoning text out

Qwen3-Omni-30B-A3B-Thinking is the Thinker-only release of Alibaba Qwen's omni-modal stack. It pairs the AuT audio encoder and a Qwen3-VL / SigLIP2-derived vision encoder with a 30B-A3B Mixture-of-Experts Transformer that consumes interleaved modality tokens under TM-RoPE positional encoding and emits text only — including an explicit reasoning trace. The Talker, MTP and Code2Wav speech-generation modules that ship with the broader Qwen3-Omni system are not part of this checkpoint.

02 · System map

From three modality lanes to one reasoning stream

Audio, image/video and text each have a dedicated front-end. Modality tokens are stitched into a single sequence under TM-RoPE — the same backbone that the Talker normally hangs off of in the Instruct system. In this Thinking checkpoint that branch is muted: the Thinker emits a thinking trace followed by the final answer in text.

03 · AuT audio encoder

20 M hours, from-scratch, 12.5 Hz token rate

The Audio Transformer (AuT) is a Qwen-trained, attention encoder-decoder pretrained from scratch on roughly 20 million hours of supervised audio. Front-end conv layers downsample the 128-mel input by 8× before 32 attention layers run with windowed attention sized 1–8 s, producing a 12.5 Hz / 80 ms-per-frame token stream that aligns 1:1 with the TM-RoPE temporal grid.

Front-end

Mel features → conv2d ↓8

16 kHz audio is windowed with a 25 ms frame and 10 ms hop and projected onto a 128-channel log-mel spectrogram. A 2-D convolutional stem downsamples the feature map by 8×, dropping the effective token rate to 12.5 Hz — one token every 80 ms.

Encoder body · config.json

32-layer attention encoder

Attention layers use dynamic windows from 1 to 8 seconds, supporting real-time prefill caching for streaming. The encoder hands off a 2048-d projection that drops directly into the Thinker's residual stream.

04 · Vision encoder

Qwen3-VL frontend initialized from SigLIP2-So400M

Image and video share a single ViT-shaped encoder. Frames are tokenized with a 16-px patch grid up to image_size 768; video is sampled dynamically and tubelet-fused at temporal_patch_size 2. Deepstack visual readouts at layers 8, 16 and 24 expose multi-scale features to the projector before they enter the Thinker.

Encoder · config.json (vision)

27-layer ViT · h=1152 · heads=16

The vision encoder is a 27-layer Transformer that follows the SigLIP2-So400M topology. After the merger it emits 2048-d tokens at half the spatial resolution and half the frame rate.

depth27 hidden_size1152 num_heads16 intermediate_size4304 patch_size16 image_size768 temporal_patch_size2 spatial_merge_size2 tokens_per_second2 out_hidden_size2048 deepstack indexes8 · 16 · 24
Video sampling

Dynamic frame sampling on the 80 ms grid

Video frames are sampled adaptively (denser for short clips, sparser for long ones) and then tubelet-fused two-at-a-time. The resulting visual tokens are emitted at tokens_per_second = 2 and stamped with TM-RoPE temporal IDs that match audio's 80 ms granularity, so audiovisual streams can be carried in a single sequence with absolute time alignment.

05 · TM-RoPE

Temporal · height · width rotary allocation

TM-RoPE splits the rotary frequency budget into three named axes — time, height and width — at a 24 / 20 / 20 allocation per head. Temporal IDs are absolute, anchored to the same 80 ms grid used by AuT, so an audio frame and the video frame it co-occurs with share a temporal coordinate. Token numbering is contiguous across modalities, letting the Thinker reason over interleaved sequences without modality-conditional position injectors.

06 · Thinker · 30B-A3B MoE

48-layer GQA Transformer with 128-expert routing

The Thinker is a Mixture-of-Experts Transformer with 48 decoder blocks, 2048-d residual stream, grouped-query attention (32 query heads, 4 KV heads, head_dim 128, QK-norm enabled), and SiLU MoE feed-forward layers. Each layer activates 8 of 128 routed experts per token, with router probabilities normalized after the top-k selection. The Thinker has no shared-expert FFN (shared_expert_intermediate_size: 0); active parameters land near 3 B per token.

Parameter budget

30 B total · ≈3 B active per token

Total parameter count refers to the Thinker MoE alone — the "A3B" portion of the name names the activated mass, not a separate Talker package. With moe_intermediate_size 768 and no shared expert (shared_expert_intermediate_size: 0), each token activates roughly 8 / 128 of the FFN parameters per layer.

07 · Thinking behavior

Sampled thinking decode, text-only output

The Thinking checkpoint produces an explicit reasoning trace and a final answer — both as text — and decodes with sampling parameters from generation_config.json. The README is explicit that Instruct models are evaluated with greedy decoding while Thinking models use the checkpoint's stochastic config, and that users should always pair the multimodal input with a written task description for best reasoning quality.

09 · Training pipeline

Three-stage pretraining + Thinker post-training

Pretraining proceeds in three stages — encoder alignment, then a general multimodal stage on ~2 T source-reported tokens, then long-context. The Thinking branch then runs SFT and a Strong-to-Weak Distillation path followed by GSPO with rule-based and model-based rewards.

Pretraining stages

Stage S1
Encoder alignment

LLM initialised from Qwen3, vision encoder from Qwen3-VL, audio encoder from AuT. Train the vision / audio adapters and encoders while the LLM is frozen.

Stage S2
General multimodal

≈ 2 T source-reported tokens. Modality mix is text-light, audio + image-heavy. Listed sub-totals sum slightly above 2 T — label as source-reported approximate.

Stage S3
Long context

Maximum token length lifted from 8 192 → 32 768. Long-form audio and video clips are upweighted to populate the extended window.

Thinker post-training

P1Supervised fine-tuning

Instruction-following SFT on curated multimodal instruction data.

P2Strong-to-Weak Distillation

Off-policy response distillation plus on-policy KL alignment against Qwen3-32B and Qwen3-235B-A22B teachers.

P3GSPO RL

Group-relative policy optimization with rule-based rewards and model-based judges — Qwen3 as the general LLM judge, Qwen2.5-VL as the visually-grounded judge.

A dedicated Thinking branch trains the model to reason explicitly over inputs from any modality. Additional Thinking-only training details beyond this description are not separately reported in the public sources.

11 · Caveats & unknowns

Things to be careful about when citing this page

Public sources for this checkpoint are the Hugging Face model card, the config and generation_config JSON, the Qwen3-Omni GitHub repository, and the technical report (arXiv 2509.17765). Where those disagree we surface the disagreement instead of choosing.

"30B-A3B" naming
Refers to the Thinker MoE scale — 30 B total / ≈ 3 B active per token — and not to an added Talker package. Adding speech-generation modules would change the parameter count.
Audio output disabled
The Thinking checkpoint sets enable_audio_output: false. Diagrams that draw a Talker, MTP or Code2Wav block as part of this checkpoint are wrong — those modules ship with the Instruct package only.
Context length nuance
The config exposes max_position_embeddings: 65 536; long-context pretraining (S3) raises the training horizon from 8 192 to 32 768; README vLLM examples use 32 768 on a single GPU and 65 536 on 4 GPUs. All three numbers are correct, in different scopes.
Vocabulary mismatch
The technical report cites 151 643 regular tokens; config.json sets vocab_size: 152 064. Treat as a source-version nuance — the LM head matrix is sized to the larger value.
Routing internals
Beyond the exposed config fields (num_experts=128, top_k=8, norm_topk_prob=true, router_aux_loss_coef=0.001) production routing behaviour — capacity factors, dispatch heuristics, expert parallelism layout — is not reported.
Active parameter exactness
"≈ 3 B active" follows the Qwen naming convention. The exact value depends on whether attention projections, embeddings and the LM head are counted as active. Cite as approximate.
Benchmark verification
Every number in §10 is taken from Qwen's own model card and report. They are not independently verified. Reproduce before quoting against other vendors' published numbers.
Thinking-only training detail
The public sources describe the Thinker post-training pipeline (SFT → Strong-to-Weak Distillation → GSPO) and note a Thinking branch that reasons over any modality. Additional Thinking-only data, hyperparameters and reward designs are not separately reported.
