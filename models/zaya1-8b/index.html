Release · 2026-05-06 · Zyphra · arXiv 2605.05365

A 0.76 B-active reasoning MoE, trained end-to-end on AMD MI300X.

ZAYA1-8B is Zyphra's reasoning-focused, Apache-2.0 mixture-of-experts language model. It is the first MoE pretrained, midtrained, and SFT'd on a full-stack AMD MI300X + Pollara 400 platform. The model has 8.4 B total parameters and routes only 0.76 B active per token through 40 decoder layers. Each block uses Compressed Convolutional Attention (CCA) in place of MLA/GQA, a three-layer MLP router with exponential-depth averaging and PID-controlled bias balancing for expert selection, and learned residual scaling on both the residual stream and the block output. Together these define Zyphra's MoE++ recipe.

01 · Forward pass

From tokens to a reasoning trace.

ZAYA1-8B is a pure text decoder — no native vision or audio tower in this release (a separate ZAYA1-VL-8B stacks a vision encoder atop this LLM). Tokens enter the Gemma3 BPE vocabulary, traverse 40 uniform decoder layers built from CCA attention and a top-1 MoE feed-forward, and exit through a tied LM head. Reasoning traces are wrapped in <think>...</think> blocks following the SFT chat template; the zaya_xml tool parser handles structured tool calls.

02 · Layer stack

40 uniform decoder layers — no F/S split, no dense warm-up.

Unlike DeepSeek-style designs that mix dense and MoE layers or alternate full and sparse attention, every one of ZAYA1-8B's 40 blocks is the same shape: CCA attention with 2× query / 8× KV compression, followed by a top-1 routed MoE feed-forward over 16 experts. The architectural variety is internal to each block (the MLP router, EDA across depth, residual scaling) rather than positional.

03 · Decoder block

Pre-norm block with residual scaling on every wire.

The block uses the standard pre-norm residual structure — RMSNorm before attention, RMSNorm before MoE, residual add after each — but with a Zyphra twist: a learned residual-scaling module sits on every residual wire and every block output, parameterized as α · x + β with α ∈ ℝ^D initialized to 1 and β initialized to 0. The attention and MoE modules are described in the deep-dive sections below.

04 · Attention

Compressed Convolutional Attention — sequence mixing in a latent space.

Where DeepSeek's MLA caches a low-rank latent c_t^KV per token and re-projects per head at attention time, CCA instead performs the sequence mixing itself in a compressed latent space via a lightweight convolutional down-projector. The result is FLOP- and memory-efficient attention with comparable cache-compression to MLA but lower prefill FLOPs and faster training throughput. In ZAYA1-8B the attention variant is reported as CCGQA with a CCA preconditioner: 2× compression on queries and 8× compression on the KV cache relative to full multi-head attention.

Attention module · all 40 layers

CCA · CCGQA · 8 Q-heads / 2 KV-heads · head dim 128

CCA is presented in the report as a FLOP- and memory-efficient attention variant that "performs sequence mixing in a compressed latent space" via a lightweight convolutional down-projector. The Apple-Silicon MLX quantization bundle confirms the on-device cache shape: per-layer standard KV plus conv_state and prev_hs, propagated as part of the CCA cache contract.

In ZAYA1-8B the variant is CCGQA (compressed-convolutional grouped-query attention): 8 query heads share 2 KV heads, head dimension is 128, and CCA preconditions the path with a 2× compression on queries and 8× on the KV cache. Position is carried by RoPE on 50% of each head's channels, leaving the other half un-rotated — a deliberate choice the report flags as preserving the no-position symmetry on half the representation.

05 · Router

ZAYA1 router — MLP, exponential-depth averaging, PID balance.

The router is where Zyphra spends its "marginal parameters." Standard MoE designs use a linear projection to score experts; ZAYA1 instead runs a three-layer GeLU MLP in a compressed 256-dim latent, after first mixing the current router representation with the previous layer's via Exponential Depth Averaging (EDA). Bias balancing is driven by a PID controller wrapped in AdamW, with the error signal being the deviation of the empirical expert-load distribution from uniform.

Expert selection · per MoE layer

r_l = W_down · x_l + γ · r_{l-1} → MLP → top-1

The MLP router replaces the linear router used in Switch / DeepSeekMoE-style models with a three-layer GeLU MLP in a down-projected 256-dim space. In the report's framing, "a small number of router parameters controls a much larger number of expert parameters, and better routing decisions significantly reduce balancing instability and improve model quality."

EDA mixes r_ℓ with the previous layer's r_{ℓ-1} via a learned scalar γ — a variant of Depth-Weighted Averaging (DenseFormer). The result is a router that "remembers" structure from earlier layers, which Zyphra argues makes balancing and expert specialization easier and shows up as faster router-load entropy recovery vs. linear routers.

PID bias balancing. Each expert has a learned bias b_{ℓ,e} added to its score before top-k. The bias gradient is the deviation of the empirical assignment probability p_{ℓ,e} from the uniform 1/E — a classical proportional-control error signal, fed through AdamW. The report describes this as "inspired by PID controllers" and notes faster convergence than the DeepSeek bias scheme.

06 · Feed-forward

16 experts, top-1, no shared or residual expert.

Every MoE layer holds 16 routed experts. Each expert is a SwiGLU-style FFN with a 2× expansion factor: 4,096 pre-activation projection / 2,048 post-activation projection (matching d_model). The router selects exactly one expert per token (k = 1) and there is no shared expert and no residual expert — unusual among contemporary MoEs. Zyphra's argument is that the more expressive router produces more confident, better-specialized routing, making additional experts in parallel less useful.

07 · Residual scaling

A 4·L·D-parameter knob on every residual wire.

The third MoE++ change is the smallest in parameter count but touches every block. A learned gate α ∈ ℝ^D and bias β ∈ ℝ^D are applied as Res-scale(x) = α ⊙ x + β, with separate (α, β) for the residual stream and for each layer's output before the residual add. The report frames this as a lower-overhead alternative to Qwen's attention gating: it lets the model down-weight parts of the residual stream and control how much prior information is retained, controlling residual-norm growth through depth without observing gradient vanishing.

Residual modulation · all 40 layers

Res-scale_res and Res-scale_out · two-place gating per block

Two separate gating pairs per block — one on the residual stream, one on the layer output. With α initialized at one and β at zero, the block starts behaving like a standard residual connection and gradually learns to downweight uninformative residual channels as training progresses.

Cost. The full bag of residual-scaling parameters is 4 · L · D per network (two pairs × two scalars × layers × hidden size). For ZAYA1-8B that is 4 · 40 · 2048 ≈ 328K parameters — well below the per-layer LayerNorm budget and negligible against the 8.4 B-parameter total.

Effect. Zyphra reports residual scaling provides "similar benefits to Qwen's attention gating scheme without the parameter or FLOP overhead of an explicit gating matrix," and helps control residual-norm growth through depth without observing gradient vanishing.

08 · Pretraining & midtraining

Four-phase reasoning-first training, all on AMD.

ZAYA1-8B is trained from scratch for reasoning: long chain-of-thought data is included from pretraining onwards via an answer-preserving (AP) trimming scheme that truncates reasoning tails while preserving the final answer at every context budget. The full pretraining-through-SFT sequence is roughly 13.86 T tokens on a 1,024-node AMD MI300X cluster, with the Muon optimizer (AdamW RMS matching).

09 · Post-training · RL cascade

SFT → four sequential RL stages, then a behavioral polish.

Post-training is organized as a four-stage verifiable-reasoning cascade followed by a fifth behavioral RL stage. The first three stages spend their compute on verifiable signals (puzzles, math, code, RLVE-Gym); chat and preference rewards arrive only at the end. All RL stages share a common spine: asynchronous PipelineRL, DPPO Binary-TV trust-region masking (δ = 0.1), Dr-GRPO SMTSN loss aggregation, MaxRL advantage estimation, and no KL in the reward. Optimizer is Muon with momentum set to zero.

10 · Test-time compute

Markovian RSA — bounded-tail recursive self-aggregation.

Markovian RSA is the model's headline test-time-compute mechanism. It combines the recursive candidate-aggregation structure of RSA (Recursive Self-Aggregation) with the bounded-workspace principle of Markovian Thinking. Each stage generates N candidates in parallel, each candidate has a bounded decode length β, and the aggregation prefill depends only on C carried-forward tails of length τ — not on the full reasoning history. ZAYA1-8B is trained for this workflow: SFT data includes aggregation examples and RL stages train both expert-model and self-aggregation variants. With a 40K / 4K configuration (40K total decode budget, 4K-token carried tail) the model reaches 91.9% AIME'25 and 89.6% HMMT'25.

Why Markovian, not full RSA
aggregation prefill depends only on C, τ — not on full reasoning history
  • Full RSA recursively aggregates all prior reasoning. Context cost grows with depth — eventually unbounded.
  • Markovian RSA aggregates only C carried tails of length τ. Workspace stays bounded regardless of how many rounds run.
  • Trained-in: SFT aggregation examples from expert-model rollouts; RL stages train both expert-model and policy-self-aggregation variants.
  • Aggregation prompt budget at training time: 20,480 tokens · sized to fit C=4 candidate tails plus the original prompt.

11 · Training stack

1,024 AMD MI300X nodes on Pollara 400 — no NVIDIA in the loop.

ZAYA1-8B is the first MoE pretrained, midtrained, and SFT-d end-to-end on an AMD Instinct MI300X stack. The cluster is a custom build with IBM, runs 1,024 MI300X nodes wired together with AMD Pensando Pollara 400 interconnect, and was validated against this 8B-MoE workload (not yet against substantially larger scales). Inference is also AMD-friendly: the model fits on a single MI300X or H100, or on consumer-grade hardware with 4-bit quantization. Released ports include Apple-Silicon MXFP4 bundles, BNB INT8 / NF4 quants, and Zyphra's own vLLM and Transformers forks.

13 · Caveats & unknowns

What this page does and does not claim.

Every numeric value in this page comes from a primary source (the official Hugging Face model card, the arXiv technical report, or the Zyphra release blog). Where a fact was not specified in those sources, it is labeled as such and not inferred. The list below collects the most material gaps and provenance notes.

Confirmed by primary sources

Architecture configuration table (40 layers, d=2048, 8/2 heads, head dim 128, CCGQA+CCA, 16 experts top-1, R=256, 50% RoPE, Gemma3 tokenizer, vocab 262,272), 0.76 B active / 8.4 B total, MoE++ recipe, training schedule (4K/8T → 4K/4T → 32K/1.2T → 131K/660B), AMD MI300X cluster (1,024 nodes, Pollara 400, IBM), RL cascade composition, Markovian RSA defaults (40K/4K), all benchmarks in §12, Apache-2.0 license, and the May 6, 2026 release date.

Inventory used a slightly different label

The local inventory describes the model as "Sparse MoE reasoning model · CCA plus GQA attention." The primary sources clarify the attention is CCGQA (a compressed-convolutional variant of GQA) with a CCA preconditioner. This page uses the source-backed naming throughout.

Active-parameter rounding

The model card uses 0.76 B active; the abstract of the technical report rounds to 700 M. We use 0.76 B for arithmetic but display "0.7 B" when matching the source convention in benchmark tables.

Embedding tying

The model card does not state whether the token embedding and LM head are tied. The standalone Apache-2.0 weight count is reported only as a total. tied to LM head: not reported.

Single-rollout AIME'25 / HMMT'25

The 91.9 / 89.6 figures shown in §10 are with the 40K/4K Markovian RSA configuration. The single-rollout values on these benchmarks are not enumerated in the model card; the side-by-side bars there are illustrative of the lift, not a verified baseline.

"GQA attention" lead from the inventory

The inventory's Attention mechanism lead: GQA resolves in primary sources to CCGQA (compressed-convolutional grouped-query). The grouping structure is real (8 Q-heads share 2 KV-heads), but the FLOP and memory profile is dominated by the CCA preconditioner, not stock GQA.

Inventory date

The inventory lists "2026-05-06" — confirmed by the Zyphra release blog ("ZAYA1-8B: Frontier intelligence density, trained on AMD", May 6, 2026) and matched by the technical report's arXiv version stamp (arXiv:2605.05365v1 [cs.AI] 06 May 2026).

Vision variant out of scope

Zyphra also released ZAYA1-VL-8B, a vision-language model built on this LLM with bidirectional image-token attention and LoRA-trained vision adapters. This page describes the text-only ZAYA1-8B; the VL variant is documented separately at huggingface.co/Zyphra/ZAYA1-VL-8B.
